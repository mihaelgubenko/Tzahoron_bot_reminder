# 📧 Настройка Email Уведомлений

## Как администратор сада будет получать уведомления о новых заявках

### Шаг 1: Создайте файл конфигурации

Скопируйте пример конфигурации:
```cmd
copy config.example.js config.js
```

### Шаг 2: Настройте email (выберите ваш провайдер)

#### Вариант A: Gmail (рекомендуется)

1. Откройте `config.js`
2. Заполните настройки:
```javascript
email: {
    host: 'smtp.gmail.com',
    port: 587,
    secure: false,
    auth: {
        user: 'ваш-email@gmail.com',
        pass: 'ваш-пароль-приложения'  // НЕ обычный пароль!
    }
}
```

3. **Создайте пароль приложения для Gmail:**
   - Перейдите: https://myaccount.google.com/security
   - Включите 2-факторную аутентификацию (если не включена)
   - Перейдите: https://myaccount.google.com/apppasswords
   - Создайте пароль приложения для "Почта" и "Другое устройство"
   - Скопируйте 16-значный пароль и вставьте в `config.js`

#### Вариант B: Outlook/Hotmail

```javascript
email: {
    host: 'smtp-mail.outlook.com',
    port: 587,
    secure: false,
    auth: {
        user: 'ваш-email@outlook.com',
        pass: 'ваш-пароль'
    }
}
```

#### Вариант C: Яндекс

```javascript
email: {
    host: 'smtp.yandex.ru',
    port: 465,
    secure: true,
    auth: {
        user: 'ваш-email@yandex.ru',
        pass: 'ваш-пароль'
    }
}
```

### Шаг 3: Укажите email администратора

В `config.js` укажите, куда отправлять уведомления:

```javascript
adminEmail: 'admin@liora-kindergarten.com',  // Ваш email
```

### Шаг 4: Перезапустите сервер

После настройки перезапустите сервер. В консоли должно появиться:
```
✅ Email server is ready to send notifications
📧 Email notifications: ENABLED (ваш-email@example.com)
```

## Что будет происходить

1. **Родитель заполняет форму** на сайте
2. **Данные сохраняются** в файлы (leads/leads.csv и JSON файлы)
3. **Администратор получает email** с информацией о заявке:
   - Имя родителя
   - Телефон (кликабельный)
   - Информация о ребенке
   - Удобное время для звонка
   - Дата и время заявки

## Тестирование

1. Заполните форму на сайте
2. Проверьте почту администратора - должно прийти уведомление
3. Проверьте консоль сервера - должно быть сообщение "📧 Email notification sent"

## Решение проблем

**Ошибка: "Email configuration error"**
- Проверьте правильность настроек в `config.js`
- Для Gmail: используйте пароль приложения, не обычный пароль
- Проверьте, что 2FA включена для Gmail

**Письма не приходят**
- Проверьте папку "Спам"
- Проверьте консоль сервера на наличие ошибок
- Убедитесь, что `config.js` существует и правильно настроен

**Письма приходят, но в спам**
- Добавьте email отправителя в контакты
- Настройте фильтры в почтовом ящике
